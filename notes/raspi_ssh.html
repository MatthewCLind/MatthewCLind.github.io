1 First Boot Procedures

1.1 Enable SSH

To enable ssh on first boot, in the root of your SD card, put a blank file called ssh (no dot no nothing)

1.2 Create new user

1.2.1 Links

https://www.rosehosting.com/blog/how-to-create-and-delete-a-user-on-debian-9/

  • $ adduser newusername
  • $ usermod -aG sudo newusername # add them to sudoers
  • $ groups newusername # veryify

1.2.2 If you want to use sudo without entering password every time

  1. Might not work, but try this:

    $ sudo visudo

    add line

    username ALL=(ALL) NOPASSWD: ALL

1.3 Delete user pi

$ deluser pi

1.4 Disable wifi and bluetooth

If you use ethernet, might as well save the power

$ sudo nano /boot/config.txt

scroll all the way down add this line

dtoverlay=pi3-disable-wifi dtoverlay=pi3-disable-bt

save it

1.5 apt-get update && upgrade

you know the drill

2.2 Set up static IP address

2.2.1 Links

https://raspberrypi.stackexchange.com/questions/37920/how-do-i-set-up-networking-wifi-static-ip-address/74428#74428 https://www.raspberrypi.org/learning/networking-lessons/rpi-static-ip-address/

2.2.2 dhcpcd method

$ nano /etc/dhcpcd.conf

scroll to bottom

add the following:

interface eth0

static ipaddress=192.168.0.2/24 static routers=192.168.0.1 static domainnameservers=192.168.0.1

interface wlan0

static ipaddress=192.168.0.2/24 static routers=192.168.0.1 static domainnameservers=192.168.0.1

2.2.3 SSH keygen configuration

  1. Links

    This is what you wanna follow if you can do it with a linux machine https://www.raspberrypi.org/documentation/remote-access/ssh/passwordless.md

  2. Make your keypair on your PC, not your server

    make sure that you never send your private key file anywhere

  3. Use Puttygen

    import your keypair if you need to you can make the keypair but then you have to convert it

  4. Put your public key on the Pi
  5. Set up Putty

    On the left side, go: Connection > SSH > Auth then at the bottom, find your private key file

    Back under Session:

    • Enter in your static IP address, keep the port 22
    • then, before your IP address, prepend username@ so, for me, it is like matthew@192.168.0.100
    • highlight Default Settings and click save
    • alternatively make a different name
  6. Change default port from 22 and disable password login
    1. Commonly used ports to avoid

      https://en.wikipedia.org/wiki/List_of_TCP_and_UDP_port_numbers

    2. Steps
      • open sshdconfig
      • $ nano /etc/ssh/sshdconfig
      • near the top, uncomment port, change port number
      • make sure you write it down and change the port in putty
      • uncomment PasswordAuthentication
      • make it say no instead of yes
